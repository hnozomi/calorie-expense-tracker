要件定義書を作成します。

## Input
- `$ARGUMENTS`: アイデアの説明やブレスト結果
- `/brainstorm` のサマリーがあれば会話コンテキストから参照

## Output
- `docs/requirements.md`

## 指示

ユーザーの入力を元に、要件定義書を作成してください。

### 作成する内容

1. **機能要件**
   - ユーザーストーリー形式: 「[ユーザー]として、[機能]をしたい。なぜなら[理由]だから。」
   - 画面ごとに整理する

2. **非機能要件**
   - パフォーマンス（読み込み速度、レスポンス時間）
   - PWA 要件（オフライン対応範囲、プッシュ通知）
   - アクセシビリティ
   - セキュリティ（該当する場合: 認証方式、データ保護の方針）

3. **データモデル案**（バックエンドが必要な場合のみ）
   - テーブル構成
   - テーブル間のリレーション
   - アクセス制御の方針

4. **API 一覧**（バックエンドが必要な場合のみ）
   - Server Actions または Route Handlers
   - 各エンドポイントの概要

5. **受入基準**
   - 各機能の完了条件
   - この基準が `/test` でのテスト設計の根拠になる

### ルール
- 不明点があればユーザーに質問して確認する
- 各セクションをユーザーに確認しながら進める
- バックエンド不要のアプリでは 3, 4 をスキップする
- 完成したら `docs/requirements.md` に保存する
- 保存後、次のステップとして `/design` を案内する
