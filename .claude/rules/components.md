# Component Conventions

## Placement
<!-- コンポーネントの配置ルール -->
- Page components → `src/app/[route]/page.tsx`
- Feature components → `src/components/features/[feature-name]/`
- Base UI components → `src/components/ui/[component-name]/`

```
src/components/
├── features/          # 機能ごとのコンポーネント
│   ├── tasks/
│   │   ├── task-card.tsx
│   │   ├── task-list.tsx
│   │   └── index.ts
│   └── auth/
│       ├── login-form.tsx
│       └── index.ts
└── ui/                # ベース UI コンポーネント
    ├── button/
    │   ├── button.tsx         # Base button (shadcn/ui)
    │   ├── submit-button.tsx  # Extends base button
    │   ├── icon-button.tsx    # Extends base button
    │   └── index.ts
    ├── card/
    │   ├── card.tsx
    │   └── index.ts
    └── input/
        ├── input.tsx
        └── index.ts
```

## UI Components
<!-- ui/ はフォルダで分ける。ベースコンポーネントを拡張して派生を作る -->
- Each UI component lives in its own folder under `src/components/ui/`.
- The base component (e.g., `button.tsx`) wraps or extends the shadcn/ui primitive.
- Derived components (e.g., `submit-button.tsx`, `icon-button.tsx`) extend the base component. Do not duplicate base styles.
- Add new shadcn/ui primitives with `pnpm dlx shadcn@latest add [component]`.

```tsx
// src/components/ui/button/button.tsx — base button (shadcn/ui)
// Generated by shadcn/ui, customized as needed

// src/components/ui/button/submit-button.tsx — extends base
import { Button } from "./button";

type SubmitButtonProps = {
  isLoading: boolean;
  children: React.ReactNode;
};

/** Form submit button with loading state */
const SubmitButton = ({ isLoading, children }: SubmitButtonProps) => {
  return (
    <Button type="submit" disabled={isLoading}>
      {isLoading ? <Spinner /> : children}
    </Button>
  );
};

export { SubmitButton };
```

```tsx
// src/components/ui/button/index.ts — barrel export
export { Button } from "./button";
export { SubmitButton } from "./submit-button";
export { IconButton } from "./icon-button";
```

## Server Components vs Client Components
<!-- Server Components をデフォルトにする。Client は必要な場合のみ -->
- Use Server Components by default.
- Add `"use client"` only when the component needs: `useState`, `useEffect`, event handlers (`onClick`, `onChange`, etc.), or browser APIs.
- Never add `"use client"` just because a parent is a Client Component — split instead.

```tsx
// Good: Server Component fetches data, Client Component handles interaction
// src/app/tasks/page.tsx (Server Component — Next.js convention uses function)
export default async function TasksPage() {
  const tasks = await fetchTasks();
  return <TaskList tasks={tasks} />;
}

// src/components/features/tasks/task-list.tsx (Client Component — arrow function)
"use client";
const TaskList = ({ tasks }: TaskListProps) => {
  const [filter, setFilter] = useState("all");
  // ...
};
```

## Next.js File Conventions
<!-- Next.js の特殊ファイルの使い方 -->
- `page.tsx` — Route page. Use `function` declaration with `export default`.
- `layout.tsx` — Shared layout wrapping child pages.
- `loading.tsx` — Loading UI shown while the page loads (Suspense boundary).
- `error.tsx` — Error UI shown when the page throws. Must be a Client Component.
- `not-found.tsx` — 404 UI for the route segment.

```tsx
// src/app/tasks/loading.tsx
export default function Loading() {
  return <Skeleton className="h-40 w-full" />;
}

// src/app/tasks/error.tsx — must be a Client Component
"use client";
export default function Error({ error, reset }: { error: Error; reset: () => void }) {
  return (
    <div>
      <p>Something went wrong.</p>
      <button onClick={reset}>Retry</button>
    </div>
  );
}

// Note: page.tsx, layout.tsx, loading.tsx, error.tsx, not-found.tsx
// use `function` declaration with `export default` (Next.js convention).
// All other components use arrow functions.
```

## Component Structure
<!-- 1コンポーネント = 1ファイル。Props 型はコンポーネントと同じファイルに定義 -->
- One component per file.
- Define Props type in the same file, directly above the component.
- Export the component as a named export from the file, and re-export via barrel (`index.ts`).

```tsx
// src/components/features/tasks/task-card.tsx
type TaskCardProps = {
  title: string;
  dueDate: Date;
  isCompleted: boolean;
};

/** Display a single task with its status and due date */
const TaskCard = ({ title, dueDate, isCompleted }: TaskCardProps) => {
  return ( ... );
};

export { TaskCard };
```

```tsx
// src/components/features/tasks/index.ts
export { TaskCard } from "./task-card";
export { TaskList } from "./task-list";
```

## Composition
<!-- コンポーネントは小さく保ち、組み合わせて使う -->
- Keep components small and focused. If a component exceeds ~100 lines, split it.
- Prefer composition over props drilling. Use children or render patterns.

```tsx
// Good: composition
<Card>
  <CardHeader>
    <CardTitle>Tasks</CardTitle>
  </CardHeader>
  <CardContent>
    <TaskList tasks={tasks} />
  </CardContent>
</Card>

// Bad: too many props
<Card title="Tasks" content={<TaskList />} headerClass="..." contentClass="..." />
```
