# 画面構成・インタラクション仕様

## 画面構成

| 画面ID | 画面名 | 主な機能 | 備考 |
| --- | --- | --- | --- |
| S-01 | ホーム（デイリービュー） | 当日の食事記録一覧・デイリーサマリー表示 | アプリ起動時のデフォルト画面 |
| S-02 | 食事登録モーダル | メニュー追加（手動/マスタ/OCR/セットメニュー） | モーダルまたはドロワー |
| S-02a | 食事アイテム編集モーダル | 登録済みアイテムのカロリー・PFC・食費の修正、削除 | ホームからアイテムタップで表示 |
| S-03 | OCRカメラ画面 | 栄養表撮影・テキスト抽出・修正フォーム | カメラAPI使用 |
| S-04 | レシピ一覧 | 登録済みレシピの検索・閲覧 | 検索機能付きカード形式 |
| S-05 | レシピ登録/編集 | レシピ詳細の入力・編集 | 材料リスト・PFC入力フォーム |
| S-06 | セットメニュー一覧 | 登録済みセットの管理・食事登録 | レシピ＋食品マスタの組み合わせ |
| S-06a | セットメニュー登録/編集 | セット名入力・アイテム追加・分量調整・合計プレビュー | レシピ/食品マスタから選択して構成 |
| S-07 | 献立カレンダー | 1週間の献立をグリッド表示 | 曜日×食事タイミングのマトリクス |
| S-08 | 食品マスタ一覧 | 登録済み食品の管理 | 検索・フィルタ機能付き |
| S-08a | 食品マスタ登録/編集 | 食品名・ブランド・カテゴリ・栄養成分・価格の入力 | OCRからの値引き継ぎ対応 |
| S-09 | ウィークリーレポート | 1週間のカロリー・PFC・食費推移 | グラフ表示 |
| S-10 | 設定 | アカウント・CSVエクスポート・データ管理 | 個人利用のためシンプル |

## 画面遷移

ボトムナビゲーション（モバイルPWA想定）で4タブ配置。

- **ホーム**（S-01）→ デイリービュー・食事記録
- **献立**（S-07）→ 1週間の献立カレンダー
- **レシピ**（S-04）→ レシピ一覧・登録
- **その他**（S-06, S-08, S-09, S-10）→ セットメニュー・食品マスタ・レポート・設定

## 画面別インタラクション仕様

### S-01: ホーム（デイリービュー）

- 日付の左右矢印タップまたは横スワイプで前後の日付に切り替え
- 各食事枠は常に表示。未登録時は「まだ登録されていません」＋「登録する」ボタン
- 登録済みの食事枠には「+ 追加する」ボタンが表示
- デイリーサマリーは食事登録・削除時にリアルタイム更新

### S-02: 食事登録モーダル

- カード方式の一括登録。複数アイテムを追加してから最後にまとめて登録
- レシピから追加する場合、1人分あたりの値に自動算出し、何人分（セレクター：±ボタン、0.5刻み）を表示
- カード各アイテムは「×」タップで個別削除可能
- カードの合計カロリー・PFC・食費はアイテム追加・削除時にリアルタイム更新
- 「○件まとめて登録する」ボタンで一括保存。カードが空の場合はボタンが非活性表示

### S-02a: 食事アイテム編集モーダル

- ホーム画面で登録済みアイテムをタップするとモーダル表示
- メニュー名・カロリー・PFC・食費の全フィールドが編集可能
- 登録元（OCR/手動/レシピ等）をバッジで表示
- 「変更を保存」で更新し、デイリーサマリーがリアルタイム再計算
- 「このアイテムを削除する」は赤色テキストで配置。タップで削除確認ダイアログを表示

### S-03: OCRカメラ画面

- カメラ起動後、栄養表を枠内に合わせて撮影ボタンをタップ
- 撮影後、OCR解析中はローディングインジケータを表示
- 解析結果はフォームにプリセット。各フィールドはタップして手動修正可能
- 「食事に登録」タップでカードに追加し、食事登録画面に戻る
- 「食品マスタに保存」タップで食品マスタ登録画面に値引き継ぎで遷移

### S-04: レシピ一覧

- 検索バーはスクロール時にトップバーに固定表示（sticky）
- レシピカードタップでレシピ詳細・編集画面に遷移
- カードの長押しで削除確認ダイアログを表示

### S-05: レシピ登録/編集

- 材料リストは「+ 材料を追加」タップで行を動的に追加
- 各材料行は左スワイプで削除ボタンを表示
- 食費合計は材料の単価×数量から自動計算・リアルタイム更新
- PFCバランスバーは入力値に連動してリアルタイム更新

### S-06: セットメニュー一覧

- 「食事に登録」タップで食事タイミング選択のダイアログを表示し、選択後にカードに全アイテムを一括追加
- 「編集」タップでセットメニュー編集画面に遷移
- セット各アイテムの構成（レシピ/食品マスタ）をバッジで視覚的に区別

### S-06a: セットメニュー登録/編集

- セットメニュー名を入力し、「レシピから追加」「食品マスタから追加」で構成アイテムを追加
- 各アイテムに分量調整（±ボタン、0.5刻み）を表示。分量変更でカロリー・PFC・食費がリアルタイム再計算
- アイテムの「×」タップで削除。最低1アイテム必要
- 合計カロリー・PFC・食費は全アイテムの値から自動計算してプレビュー表示

### S-07: 献立カレンダー

- 横軸に日付・曜日、縦軸に朝食・昼食・夕食を配置したグリッドレイアウト
- グリッドは横スクロール対応。当日の列を初期表示位置とする
- 空セル（「+」表示）タップでメニュー選択モーダルを表示
- 登録済みセルタップで変更・削除メニューを表示
- 週間合計はセル追加・変更時にリアルタイム更新
- 前後の週への移動は左右矢印タップ

### S-08: 食品マスタ一覧

- 検索バーは食品名・ブランド名の両方で検索可能
- カードタップで食品マスタ編集画面に遷移
- カードの長押しで削除確認ダイアログを表示

### S-08a: 食品マスタ登録/編集

- 食品名（必須）・ブランド名・カテゴリ（ドロップ選択）・栄養成分・デフォルト価格・メモを入力
- 「栄養表から読み取る」ショートカットタップでOCRカメラ画面に遷移し、読み取り結果を自動入力
- 食品マスタ一覧の「+ 新規」またはOCR画面の「食品マスタに保存」から遷移

### S-09: ウィークリーレポート

- 前後の週への移動は左右矢印タップ
- 棒グラフの各バーをタップすると、該当日のデイリービューに遷移
- PFCバランスバーはタップで詳細の表示/非表示を切り替え

### S-10: 設定

- CSVエクスポートは食事記録・レシピ・食品マスタ・セットメニューの4種類を個別にダウンロード可能
- 「全データを削除」は赤色の危険ゾーンに配置。タップで二段階確認ダイアログを表示（「全データ削除」と入力して確認）
- 「キャッシュをクリア」実行後、トースト通知で完了を表示

## 共通UIパターン

### ローディング状態

- 画面初回読み込み時：カード型のスケルトンスクリーン（灰色のプレースホルダーがパルスアニメーション）
- ボタン操作時（登録・保存・削除）：ボタン内にスピナーを表示し、ボタンを非活性化（二重送信防止）
- OCR解析中：全画面オーバーレイ（中央にスピナー＋「読み取り中...」テキスト）

### エラー状態

- API通信エラー：画面上部にトースト通知（赤背景・白文字）＋リトライボタン
- オフライン検知時：画面上部に固定バー「オフラインです。一部機能が制限されます」
- バリデーションエラー：該当フィールドの枠線を赤色に変更し、フィールド下にエラーメッセージ

### 認証エラー状態

- **ログイン失敗**（メール/パスワード不一致）：フォーム上部にインラインエラー「メールアドレスまたはパスワードが正しくありません」
- **サインアップ失敗**（メール重複）：フォーム上部にインラインエラー「このメールアドレスは既に登録されています」
- **サインアップ失敗**（パスワード要件不足）：パスワードフィールド下にエラー「パスワードは8文字以上で入力してください」
- **セッション期限切れ**：自動的に `/login` にリダイレクトし、トースト通知「セッションが期限切れになりました。再度ログインしてください」
- **ネットワークエラー**（認証中）：フォーム上部にエラー「通信エラーが発生しました」＋リトライ可能な状態を維持（入力値を保持）

### 空状態（Empty State）

- 一覧画面が0件の場合：中央にアイコン＋説明テキスト＋CTAボタンを配置
- ホーム画面で全食事枠が未登録の場合：サマリーエリアにプレースホルダーメッセージ
- 検索結果が0件の場合：「該当する結果が見つかりませんでした」

### 操作完了フィードバック

- 成功時：画面上部に緑色トースト通知（例：「3件の食事を登録しました」）2秒間表示後、自動非表示
- 削除時：「削除しました」トースト表示。トースト内に「元に戻す」リンクを配置（5秒以内に取り消し可能）

### 削除確認ダイアログ

- 統一パターン：モーダルダイアログ（背景暗転）＋「〇〇を削除しますか？」＋「キャンセル」（グレー）/「削除」（赤）ボタン
- レシピ・食品マスタ・セットメニューの削除は一覧画面でカードを長押しして発火
- 食事アイテムの削除はアイテム編集モーダル内の「このアイテムを削除する」から発火
- 「全データを削除」（設定画面）のみ二段階確認（テキスト入力による確認を要求）

### 献立→食事記録への転記フロー

- ホーム画面にて、当日の献立が存在＆未転記の場合、サマリーカードの下に「今日の献立を転記する」バナーを表示
- バナーをタップで食事登録画面（カード方式）に遷移。献立の全メニューがカードにプリセットされた状態で表示
- ユーザーはカード内容を確認・修正してから「まとめて登録する」で確定
- 転記後、献立カレンダーの該当セルに「記録済み」バッジ（チェックマーク）を表示
